Python介绍

Python的设计哲学是“优雅”、“明确”、“简单”。在设计Python语言时，如果面临多种选择，Python开发者一般会拒绝花俏的语法，而选择明确的没有或者很少有歧义的语法。由于这种设计观念的差异，Python源代码通常被认为具备更好的可读性，并且能够支撑大规模的软件开发。这些准则被称为Python格言。在Python解释器内运行import this可以获得完整的列表。

“对于一个特定的问题，只要有一种最好的方法来解决”
这在由Tim Peters写的Python格言（称为The Zen of Python）里面表述为：There should be one-and preferably only one-obvious way to do it。

Python是一门相较于其他语言比较好入门的语言，并且也是当前人工智能相关领域的主流语言。Python是一门面向对象编程 的语言，Python在各个编程语言中比较适合新手学习，Python解释器易于扩展，可以使用C、C++或其他可以通过C调用的语言扩展新的功能和数据类型。 Python也可用于可定制化软件中的扩展程序语言。Python丰富的标准库，提供了适用于各个主要系统平台的源码或机器码。

Python秉承着开源精神，一个显著的特征就是有很多包、库可以进行导入并且调用，也可以导入自己写的程序包进行调用，出名的包不仅能让代码更加规范，也让进行一些训练任务更加简洁方便。

1.2 变量的规则

但是变量的命名是有一定规则的，我们先列出所有规则，然后一一举例：

  • 变量名只能包含字⺟、数字和下划线 。变量名能以字⺟或下划线打 头，但不能以数字打头。例如，可将变量命名为 message_1，但不能 将其命名为 1_message。
  • 变量名不能包含空格，但能使⽤下划线来分隔其中的单词。例如，变 量名 greeting_message 可⾏，但变量名 greeting message 会 引发错误。
  • 不要将 Python 关键字和函数名⽤作变量名。例如，不要将 print ⽤作 变量名，因为它被 Python 留作特殊⽤途（请参⻅附录 A.4）。
  • 变量名应既简短⼜具有描述性。例如，name ⽐ n 好， student_name ⽐ s_n 好，name_length ⽐ length_of_persons_name 好。
  • 慎⽤⼩写字⺟ l 和⼤写字⺟ O，因为它们可能被⼈错看成数字 1 和 0。
    ps：在Python3中，变量名甚至可以用中文，但是不推荐这样做，应该创建通俗易懂的变量名，比如利用英文翻译你的中文来作为变量名。

    以上参考《Python编程：从入门到实践》

赋值运算符

赋值运算符时最常见的运算符，无论是前面的变量赋值还是后面的运算赋值，都是必不可少的运算符。
赋值运算符中有一类叫复合赋值运算符，可以理解为在赋值前进行一次运算

比较运算符和逻辑运算符

比较运算符也称为关系运算符，包括==、!=、<、>、<=、>=，就是用来比较符号两边的关系。注意其中的==，这是两个等号，用来比较两者是否相等，请与赋值运算符区别。比较运算符的返回结果是一个布尔值，True或者False。

逻辑运算符有三个，分别为and,or,not。相信学过其他语言的同学都会知道是什么意思，and需要两边的结果的布尔值都是True结果才为True，否则就是False。or则只需有一个布尔值为True结果就为True，否则就是False。而not则是取反，也就是说，后面的布尔值如果是True运算结果就是False，而后面的布尔值如果是False则运算结果就是True。

ps：当and的左边布尔值为False，不管右边的值，将会直接返回False，跳过处理右边。（这叫短路处理）。同理or也是有短路处理的功能的，当左边为True时，会直接跳过右边的表达式。

3.2 浮点数

Python中浮点数就是带小数点的数，小数点可以出现在数的任何位置上，以表示任何数。
当你使用浮点数时，你可以不用考虑其他行为，python通常会按照你的预期的方式进行处理，所以Python是世界上最好的语言！！！（/doge）

如下图，可以看到当你把一个整数加上小数点，就变成了一个浮点数，这就是**小数点可以出现在任何位置上的原因。

但是需要注意的是，当浮点数参与运算时，小数位数可能是不确定的，可以理解为浮点数其实是计算机给你的一个精确值，但不是绝对和运算结果相等的，可以参考下图。
这是所有编程语言都存在的问题，说白了这是计算机硬件处理和储存的问题，具体可参考计算机组成原理，这里不详细解释，所以计算机会忽略多余的小数位数来尽力精确的表示结果。

  • 两个整数相除，结果也是一个浮点数，即使这两个数可以整除。
  • 在一系列运算中，若存在一个浮点数，那么结果也是浮点数。

3.3 常量

常量就是在程序的整个生命周期内都保持不变的变量。Python没有内置的常量类型，但是一般会使用全大写字母的变量名来指出某个变量为常量，其值应始终保持不变

4. 注释

注释是让代码逻辑清晰的一个很有用的功能，注释是不会被程序执行的。随着程序越来越大，增加注释来对代码进行大致阐述，可以让别人看你代码时更加容易理解。

编写方式：井号#来标识。#后的内容都会被Python解释器忽略

编写注释的主要⽬的是阐述代码要做什么，以及是如何做的。在开发项⽬ 期间，你对各个部分如何协同⼯作了如指掌，但是⼀段时间过后，你可能会忘记⼀些细节。当然，你总是可以通过研究代码来确定各个部分的⼯作 原理，但通过编写注释以清晰的⾃然语⾔对解决⽅案进⾏概述，可以节省很多时间。
要成为专业程序员或与其他程序员合作，就必须编写有意义的注释。当前，⼤多数软件是合作编写⽽成的，编写者既可能是同⼀家公司的多名员⼯，也可能是众多致⼒于同⼀个开源项⽬的⼈。训练有素的程序员都希望代码中包含注释，因此你最好从现在开始就在程序中添加描述性注释。新⼿程序员最值得养成的习惯之⼀就是，在代码中编写清晰、简洁的注释。 如果不确定是否要编写注释，就问问⾃⼰：在找到合理的解决⽅案之前， 考虑了多个解决⽅案吗？如果答案是肯定的，就编写注释，对你的解决⽅案进⾏说明吧。相⽐以后再回头添加注释，删除多余的注释要容易得多。 从现在开始，本书的⽰例都将使⽤注释来阐述代码的⼯作原理。

如果你想写shi山代码除外，网上很多说不写注释加上乱七八糟的变量名可以增加自己的不可替代性。这边建议不如多学学高门槛的知识，来增加自己的不可代替性。
